## 第一次接触：Windows 上的 Trae

第一次用 Trae 还是在 Windows 时代。当时在论坛上看到有人推荐，抱着试试看的心态装了一下，发现 AI 辅助确实比 VSCode 爽快不少。那段时间用它写了不少前端代码，整体体验还不错。印象比较深的是那时候 Trae 的图标还不是现在这个对话框样式。

## Ubuntu 时期：回归 VSCode

后来换成了 Ubuntu 做主力系统，翻了一圈发现 Trae 还没有 Linux 版本。虽说可以用 Wine 之类的跑起来，但总觉得不够原生，索性就换回了 VSCode。

## Debian 13：重新相遇 Trae

最近把系统换成了 Debian 13，抱着试试看的心态又去搜了一下 Trae，发现已经出了 Linux 版本。装好之后用了几天，有几点比较明显的感受：

- **AI 辅助**确实好用，补全和代码解释比 VSCode 默认的要强不少，对写代码效率有帮助
- **中文支持**对国内开发者友好，界面和文档都不需要额外折腾

## 一些个人感受

唯一的遗憾是 **Trae Solo 目前还没有 Linux 版本**。作为一个已经在 Linux 上扎根的用户，只能眼巴巴看着 Windows 和 macOS 用户体验 Solo，略感羡慕。希望 Trae 团队能早日把 Solo 也带到 Linux 平台上来。

另外想吐槽的是 Trae 似乎不支持 GitHub Workspace，这也是目前换回 Trae 的一个障碍。
